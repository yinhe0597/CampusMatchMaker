### 2.2 环境配置步骤

#### 步骤 1：克隆仓库

```bash
git clone <repository-url>
cd time_tag_peidui
```

#### 步骤 2：配置环境变量

```bash
cp .env.example .env
# 编辑 .env，填入本地配置（数据库密码等）
```

#### 步骤 3：启动 MySQL（使用 Docker）

```bash
docker compose up -d mysql
```

或使用本地 MySQL，确保创建数据库：

```sql
CREATE DATABASE campus_collab DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 步骤 4：执行数据库迁移

```bash
cd server
migrate -path migrations -database "mysql://root:password@tcp(localhost:3306)/campus_collab" up
```

#### 步骤 5：安装后端依赖

```bash
cd server
go mod download
```

#### 步骤 6：安装前端依赖

```bash
cd web
pnpm install
```

## 4. 环境变量配置

### 4.1 `.env.example` 模板

```bash
# ===== 服务配置 =====
APP_NAME=campus_collab
APP_ENV=development          # development / staging / production
APP_PORT=8080
APP_HOST=0.0.0.0

# ===== 数据库配置 =====
DB_HOST=localhost
DB_PORT=3306
DB_NAME=campus_collab
DB_USER=root
DB_PASSWORD=password
DB_MAX_IDLE_CONNS=10
DB_MAX_OPEN_CONNS=100

# ===== JWT 配置 =====
JWT_SECRET=your-secret-key-change-in-production
JWT_EXPIRE_HOURS=24
JWT_REFRESH_HOURS=168

# ===== 加密密钥（用于学号、手机号加密） =====
ENCRYPT_KEY=your-32-byte-aes-key-change-me-now

# ===== Redis（可选） =====
REDIS_ENABLED=false
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# ===== 日志配置 =====
LOG_LEVEL=debug              # debug / info / warn / error
LOG_FORMAT=json              # json / console

# ===== CORS =====
CORS_ORIGINS=http://localhost:5173
```

### 4.2 安全提醒

- `.env` 文件已加入 `.gitignore`，**禁止提交到仓库**
- 生产环境的 `JWT_SECRET` 和 `ENCRYPT_KEY` 必须替换为随机强密钥
- 配置变更通过环境变量注入，**禁止硬编码到代码中**

```yaml
# docker-compose.yml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
      MYSQL_DATABASE: ${DB_NAME}
    ports:
      - "${DB_PORT}:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  server:
    build:
      context: ./server
      dockerfile: Dockerfile
    ports:
      - "${APP_PORT}:8080"
    depends_on:
      mysql:
        condition: service_healthy
    environment:
      - APP_ENV=${APP_ENV}
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_NAME=${DB_NAME}
      - DB_USER=root
      - DB_PASSWORD=${DB_PASSWORD}
      - JWT_SECRET=${JWT_SECRET}
      - ENCRYPT_KEY=${ENCRYPT_KEY}

  web:
    build:
      context: ./web
      dockerfile: Dockerfile
    ports:
      - "3000:80"
    depends_on:
      - server

volumes:
  mysql_data:
```

## 6. CI/CD 流水线

### 6.1 CI 阶段

```yaml
# .github/workflows/ci.yml（示例）
name: CI

on:
  push:
    branches: [develop, main]
  pull_request:
    branches: [develop]

jobs:
  backend:
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: test
          MYSQL_DATABASE: campus_collab_test
        ports:
          - 3306:3306
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.21'
      - name: Lint
        run: |
          cd server
          go vet ./...
      - name: Test
        run: |
          cd server
          go test -race -coverprofile=coverage.out ./internal/...
      - name: Build
        run: |
          cd server
          go build -o /dev/null cmd/main.go

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'pnpm'
          cache-dependency-path: web/pnpm-lock.yaml
      - name: Install & Lint & Build
        run: |
          cd web
          pnpm install --frozen-lockfile
          pnpm lint
          pnpm build
```

### 7.4 原子能力层关键测试用例

根据开发文档要求，以下场景必须有覆盖：

**空闲时段计算引擎：**
- 无人有空 → 返回空列表
- 全员有空 → rate = 1.0
- 部分重叠 → 只返回交集部分
- 跨午休时段 → 按配置过滤
- 临界时间点 → 边界正确
- 单人场景 → 直接返回空闲

**课表继承逻辑：**
- 首次录入 → 正确写入公共课表
- 自动继承 → 后续用户获得完整课表
- 选修补充 → 个人课表正确叠加
- 冲突覆盖 → is_overridden 正确标记
- 协作纠错 → 纠错流程状态机正确

## 8. 日志与监控规范

### 8.1 日志格式

使用 JSON 结构化日志（Zap），每条日志包含：

```json
{
  "level": "info",
  "ts": "2025-09-15T10:30:00.000Z",
  "caller": "handler/poll_handler.go:42",
  "msg": "poll created",
  "request_id": "abc-123-def",
  "user_id": 1,
  "poll_id": 5,
  "scope_type": "class",
  "scope_id": 1
}
```

### 8.2 日志级别使用规范

| 级别 | 使用场景 |
|------|---------|
| DEBUG | 开发调试信息，生产环境关闭 |
| INFO | 关键业务操作（创建投票、提交课表等） |
| WARN | 异常但可恢复的情况（重试、降级） |
| ERROR | 需要人工介入的错误 |

### 8.3 敏感信息脱敏规则

日志中**禁止记录**以下信息：
- 用户密码 / Token
- 完整学号（只显示前2后2位，中间用 `*` 替代）
- 完整手机号（只显示前3后4位）
- 他人课表详情

```go
// 脱敏示例
func maskStudentID(id string) string {
    if len(id) <= 4 {
        return "****"
    }
    return id[:2] + strings.Repeat("*", len(id)-4) + id[len(id)-2:]
}
// "2024001234" → "20****34"
```

### 8.4 请求追踪

- 每个 HTTP 请求分配或透传 `X-Request-ID`
- 同一请求链路上的所有日志携带相同 `request_id`
- 便于排查问题时追踪完整请求路径
