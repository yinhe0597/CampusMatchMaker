## 1. Go 后端编码规范

### 1.1 命名规范

#### 包名

- 全小写，短而明确
- 不使用下划线或驼峰

```go
// ✅ 正确
package engine
package handler
package model
package timeslot

// ❌ 错误
package timeSlot
package user_model
package handlers
```

#### 结构体与接口

```go
// 结构体：PascalCase
type TimeSlotEngine struct { ... }
type PollService struct { ... }
type User struct { ... }

// 接口：以 er 结尾或行为描述
type Calculator interface { ... }
type SlotProvider interface { ... }
type Repository interface { ... }
```

#### 变量与函数

```go
// 导出函数/方法：PascalCase
func NewTimeSlotEngine() *TimeSlotEngine { ... }
func (e *TimeSlotEngine) CalculateFreeSlots(...) []FreeSlotResult { ... }

// 私有函数/方法：camelCase
func (e *TimeSlotEngine) mergeOccupiedSlots(slots []OccupiedSlot) []OccupiedSlot { ... }
func normalizeTime(minutes int) int { ... }

// 布尔变量：is/has/can 前缀
isActive := true
hasPermission := checkPerm(userID)

// 数量/计数：使用 Count 后缀
votedCount := 15
memberCount := 30
```

#### 常量

```go
// 常量组
const (
    StatusActive  = 1
    StatusDisabled = 0

    RoleOwner  = "owner"
    RoleAdmin  = "admin"
    RoleMember = "member"
)

// 或使用 UPPER_SNAKE_CASE
const MAX_RETRY_COUNT = 3
```

每个 Go 文件遵循以下顺序组织：

```go
// 4. 构造函数

// 5. 公开方法

// 6. 私有方法

```

### 1.3 依赖注入

使用构造函数注入依赖，不使用全局变量：

```go
// ✅ 正确：构造函数注入
func NewPollService(repo PollRepository, engine *timeslot.Engine) *PollService {
    return &PollService{repo: repo, engine: engine}

// ❌ 错误：全局变量
var pollRepo PollRepository
var engine *timeslot.Engine
```

### 1.4 HTTP Handler 规范

```go
// Handler 函数结构：参数校验 → 调用 Service → 组装响应
func (h *PollHandler) CreatePoll(c *gin.Context) {
    // 1. 参数校验
    var req CreatePollRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.BadRequest(c, "参数错误")
        return
    }

    // 2. 获取当前用户
    userID := c.GetUint("user_id")

    // 3. 调用 Service
    poll, err := h.service.CreatePoll(c.Request.Context(), userID, req)
    if err != nil {
        response.HandleError(c, err)
        return
    }

    // 4. 组装响应
    response.Created(c, poll)
}
```

## 2. Vue3 前端编码规范

### 2.1 组件命名

- 使用 **PascalCase**，至少两个单词组合
- 页面组件以 `View` 结尾
- 表单组件以 `Form` 结尾

```javascript
// ✅ 正确
PollCreateView.vue
RecommendedSlots.vue
TimeSlotCard.vue
VotePanel.vue

// ❌ 错误
poll.vue          // 太短，非 PascalCase
create.vue        // 单字
poll-create.vue   // kebab-case
```

### 2.2 目录约定

| 目录 | 用途 | 说明 |
|------|------|------|
| `views/` | 页面级组件 | 对应路由，按业务域分子目录 |
| `components/` | 可复用组件 | 按业务域和通用分子目录 |
| `composables/` | 组合式函数 | 逻辑复用，命名 `useXxx` |
| `stores/` | Pinia Store | 按业务域拆分 |
| `api/` | API 调用层 | 与后端模块对应 |
| `utils/` | 工具函数 | 纯函数，无副作用 |
| `assets/` | 静态资源 | 图片、字体、全局样式 |

### 2.4 Pinia Store 规范

```javascript
// stores/poll.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { pollApi } from '@/api/poll'

export const usePollStore = defineStore('poll', () => {
  // 状态
  const polls = ref([])
  const currentPoll = ref(null)
  const loading = ref(false)

  // 计算属性
  const openPolls = computed(() =>
    polls.value.filter(p => p.status === 'open')
  )

  // Actions
  async function fetchPolls(params) {
    loading.value = true
    try {
      const res = await pollApi.list(params)
      polls.value = res.data.list
    } finally {
      loading.value = false
    }
  }

  async function getPoll(id) {
    const res = await pollApi.detail(id)
    currentPoll.value = res.data
    return res.data
  }

  return { polls, currentPoll, loading, openPolls, fetchPolls, getPoll }
})
```

### 2.5 API 调用层规范

```javascript
// api/request.js - Axios 封装

request.interceptors.response.use(
  response => response.data,

      // Token 过期，跳转登录
      const userStore = useUserStore()
      userStore.logout()
      window.location.href = '/login'
    }
    return Promise.reject(new Error(message || '网络错误'))
  }
)
```

## 3. 注释与文档规范

### 3.1 Go 注释规范

```go
    }

    // 第二步：对每个用户取反，得到空闲时段
    // 时间轴范围由 config.DayStart 和 config.DayEnd 决定
    // ...
}
```

### 3.2 注释原则

| 场景 | 要求 |
|------|------|
| 公开函数/方法 | 必须有 godoc 注释（`// FuncName ...`） |
| 复杂算法 | 必须有行内注释解释**为什么**，而非**做了什么** |
| 隐私相关逻辑 | 必须注释说明隐私保障措施 |
| 业务决策 | 注释说明决策理由，便于日后回溯 |

### 3.3 禁止的注释

```go
// ❌ 描述代码做了什么（代码本身已足够清楚）
// 遍历 schedules 数组
for _, s := range schedules { ... }

// ❌ 注释掉的废弃代码
// oldFunc()

// ❌ TODO 没有关联任务编号
// TODO: 优化性能
```

## 4. 分层代码规范（Go 后端）

### 4.1 Handler 层

```go
// 职责：参数校验、调用 Service、组装响应
// 禁止：包含业务逻辑、直接操作数据库、调用 Engine

type PollHandler struct {
    service *service.PollService
}

func (h *PollHandler) CreatePoll(c *gin.Context) {
    var req CreatePollRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.BadRequest(c, "参数错误: "+err.Error())
        return
    }
    userID := c.GetUint("user_id")
    result, err := h.service.CreatePoll(c.Request.Context(), userID, req)
    if err != nil {
        response.HandleError(c, err)
        return
    }
    response.Created(c, result)
}
```

### 4.2 Service 层

```go
// 职责：业务编排，调用 Repository 和 Engine
// 禁止：直接处理 HTTP 请求/响应

type PollService struct {
    repo       PollRepository
    engine     *timeslot.Engine
    ttService  *TimetableService
}

func (s *PollService) CreatePoll(ctx context.Context, userID uint, req CreatePollRequest) (*Poll, error) {
    // 1. 获取参与者列表
    members, err := s.repo.GetScopeMembers(ctx, req.ScopeType, req.ScopeID)
    if err != nil {
        return nil, err
    }

    // 2. 获取参与者课表，转换为引擎输入格式
    userIDs := extractUserIDs(members)
    schedules, err := s.ttService.GetUserSchedules(ctx, userIDs)
    if err != nil {
        return nil, err
    }

    // 3. 调用引擎计算推荐时段
    engineInput := convertToEngineFormat(schedules)
    freeSlots := s.engine.CalculateFreeSlots(engineInput, timeslot.DefaultConfig())

    // 4. 创建投票和选项
    poll, err := s.repo.CreatePoll(ctx, userID, req)
    if err != nil {
        return nil, err
    }

    // 5. 保存推荐选项
    options := convertToOptions(poll.ID, freeSlots)
    if err := s.repo.CreateOptions(ctx, options); err != nil {
        return nil, err
    }

    return poll, nil
}
```

### 4.4 Engine 层（原子能力）

```go
// 职责：纯计算，无副作用
// 禁止：import domain/workflow/handler 包、读写数据库、调用外部 API
// 特征：相同输入永远得到相同输出

// CalculateFreeSlots 计算共同空闲时段（纯函数）
func CalculateFreeSlots(schedules []UserSchedule, config EngineConfig) []FreeSlotResult {
    if len(schedules) == 0 {
        return nil
    }

    // ① 合并每个用户的占用时段
    mergedSchedules := make([][]OccupiedSlot, len(schedules))
    for i, s := range schedules {
        mergedSchedules[i] = mergeOverlapping(s.Slots)
    }

    // ② 取反得空闲时段
    freeSchedules := make([][]FreeSlot, len(mergedSchedules))
    for i, occupied := range mergedSchedules {
        freeSchedules[i] = invertSlots(occupied, config)
    }

    // ③ 多人求交集
    intersection := intersectFreeSlots(freeSchedules)

    // ④ 聚合统计
    results := aggregateResults(intersection, len(schedules))

    // ⑤ 按参与率降序排序
    sort.Slice(results, func(i, j int) bool {
        return results[i].Rate > results[j].Rate
    })

    return results
}
```

## 5. 错误处理规范

### 5.1 自定义错误类型

```go
// pkg/errors/errors.go
package errors

type AppError struct {
    Code    int    // 业务错误码
    Message string // 用户可见的错误信息
    Err     error  // 原始错误（用于日志）
}

func (e *AppError) Error() string {
    if e.Err != nil {
        return e.Err.Error()
    }
    return e.Message
}

func New(code int, message string) *AppError {
    return &AppError{Code: code, Message: message}
}

func Wrap(code int, message string, err error) *AppError {
    return &AppError{Code: code, Message: message, Err: err}
}

// 预定义错误
var (
    ErrUnauthorized = New(1100, "未登录或登录已过期")
    ErrForbidden    = New(1106, "无操作权限")
    ErrNotFound     = New(1002, "资源不存在")
)
```

### 5.2 错误处理原则

| 原则 | 说明 |
|------|------|
| 不吞错误 | 每个错误必须处理或向上返回，禁止 `_ = err` |
| 日志记完整 | 日志中记录完整错误栈，API 响应只返回友好信息 |
| 上下文附加 | 向上返回时使用 `fmt.Errorf("context: %w", err)` 附加上下文 |
| 分类处理 | Handler 层根据 AppError.Code 返回对应 HTTP 状态码 |

```go
// ✅ 正确：附加上下文向上返回
func (s *PollService) GetPoll(ctx context.Context, id uint) (*Poll, error) {
    poll, err := s.repo.GetPollByID(ctx, id)
    if err != nil {
        return nil, fmt.Errorf("get poll %d: %w", id, err)
    }
    if poll == nil {
        return nil, errors.New(1400, "投票不存在")
    }
    return poll, nil
}

// ❌ 错误：吞掉错误
func (s *PollService) GetPoll(ctx context.Context, id uint) *Poll {
    poll, _ := s.repo.GetPollByID(ctx, id)
    return poll
}
```

### 6.2 加载顺序（优先级从高到低）

1. 环境变量（最高优先级）
2. `configs/config.yaml`（本地配置）
3. `configs/config.example.yaml`（默认值）

### 6.3 配置使用规则

- **禁止硬编码**：所有可变配置必须通过配置文件或环境变量管理
- **敏感信息**：密码、密钥等只通过环境变量传入，不写入配置文件
- **环境隔离**：不同环境使用不同配置文件，通过 `APP_ENV` 环境变量切换
- **配置验证**：启动时校验必要配置项，缺失则 panic 并提示

```go
// 配置加载示例
func LoadConfig() (*Config, error) {
    viper.SetConfigName("config")
    viper.SetConfigType("yaml")
    viper.AddConfigPath("./configs")
    viper.AutomaticEnv() // 环境变量覆盖

    if err := viper.ReadInConfig(); err != nil {
        return nil, fmt.Errorf("read config: %w", err)
    }

    var cfg Config
    if err := viper.Unmarshal(&cfg); err != nil {
        return nil, fmt.Errorf("unmarshal config: %w", err)
    }

    // 校验必要配置
    if cfg.JWT.Secret == "" {
        return nil, fmt.Errorf("JWT_SECRET is required")
    }

    return &cfg, nil
}
```
