## 四、技术决策记录

### 4.1 学号存储方案（已调整）

**原方案**：AES-256-CBC 加密存储
**问题**：AES-CBC 使用随机 IV，每次加密结果不同，无法用于数据库查找
**现方案**：
- `users.student_id` → SHA-256 哈希存储（确定性，可查找，不可逆推）
- `student_auth.student_id` → 明文存储（仅用于认证审核流程）
- JWT Claims 中存储原始学号（短期有效，用于请求链路）

**影响**：
- `users.student_id` 列从 `VARCHAR(30)` 改为 `VARCHAR(256)`
- `student_auth.student_id` 列同步改为 `VARCHAR(256)`
- 数据库已通过 ALTER TABLE 修改，迁移文件待同步更新

- 实际使用：Go 1.25.1（go.mod 中声明）
